# ARIA-H 정성검증 항목별 세부 평가 지침

## 2.1 임상적 도입 가치

의료진이 유상 도입 가치가 있는 성능 수준인지 확인

| 세부 항목 | 확인 사항 | 평가 방법 | 수집 대상 영상 |
|---|---|---|---|
| 검출 성능 충분성 | 등급 경계(4→5, 9→10개)에서 잘 맞는지 | 등급 경계 개수 case 확인 | 등급 경계 개수 병변 보유 case |
| 시퀀스별 수용성 | SWI·GRE 에서 검출 차이가 없는지 | 시퀀스별 분리 분석 | SWI·GRE 동시 촬영 case |
| 종단 수용성 | 병변 신생·증감을 신뢰성 있게 반영하는지 | 종단 검출 비교 | 종단(추적) 영상 보유 case |
| 판독 부담 절감 | 단일, 다량, 소형 CMB 및 타 출혈 소견 동반 시 민감도 특이도가 보존되는지 | 해당 case 확인 | CMB 1개, 10개 이상, 소형 CMB, 타 출혈 동반 case |

**등급 기준**

| 등급 | 기준 |
|---|---|
| Good | 도입 가치 명확. 등급 경계 정확도 높고 개수 상관 없이 민감, 특이도 보존|
| Moderate | 도입 가치는 있으나 특정 시퀀스·병변 크기, 개수에 제한적.|
| Warning | 현 성능으로 도입 가치 불확실. 개선 후 재평가. |
| Critical | 도입 가치 없거나 판독 품질 저해 우려. |

## 2.2 오류 케이스 임상 위해도

분석 실패·오류가 환자에게 미치는 위해 가능성이 허용 가능한 수준인지 확인

| 세부 항목 | 확인 사항 | 평가 방법 | 수집 대상 영상 |
|---|---|---|---|
| 위음성 위해 | 미검출로 중증도 과소평가를 유발하는지 | 위음성 케이스 등급 영향 분석 | 위음성 개수별 case |
| 위양성 위해 | mimic 오검출로 불필요한 중증도 과대평가를 유발하는지 | 위양성 케이스 등급 영향 분석 | 위양성 개수별 case, mimic 종류별 case |
| 고위해 소견 처리 | macrohemorrhage 등 즉각 조치 소견시 어떻게 출력하는지 | 고위험 케이스 별도 검토 | Macrohemorrhage 동반 case |

**등급 기준**

| 등급 | 기준 |
|---|---|
| Good | 오류 발생 가능성 낮고, 특히 고위험 누락 없음. |
| Moderate | 경미한 오류 가능성 있으나 의학적으로 혀용 가능한 수준. 모니터링 필요. |
| Warning | 특정 오류(등급 경계 위음성·반복 위양성)가 치료 결정에 영향. 개선 후 재평가. |
| Critical | 고위험 소견 누락 등 환자 안전에 중대한 위해 우려. Release 보류 필요. |

## 2.3 출력 결과 타당성

의학적으로 납득 가능한 위치·크기·개수·형태의 마스크를 출력하는지 확인

| 세부 항목 | 확인 사항 | 평가 방법 | 수집 대상 영상 |
|---|---|---|---|
| 위치 타당성 | 뇌 외부·artifact 오검출이 없는지 | box overlay 해부학적 검토 | 뇌 외부 오검출 case |
| 크기 타당성 | box가 병변 대비 너무 크거나 작지 않은지| 병변 직경 대비 box 적합성 측정 | 단일 슬라이스 내 CMB ≥5개 case |
| 개수 타당성 | 검출 개수가 납득 가능하고 중증도 등급에 잘 반영되는지 | 다량 CMB case 개수 분포·등급 일치 검토 | CMB ≥20개 case|
| 형태 타당성 | round-to-oval 저신호·blooming 특성 병변을 검출하는지 | 검출 병변 형태 적합성 검토 | 검토하는 모든 CMB 양성 case |
| Mimic 감별 | 혈관 flow void·석회화·cavernoma 등과 감별하는지 | mimic 케이스 오검출 검토 | mimic 오검출 case |

**등급 기준**

| 등급 | 기준 |
|---|---|
| Good | 위치·크기·개수·형태 모두 타당. mimic 감별 양호. |
| Moderate | 대체로 타당하나 경미한 box 편차·소수 mimic 혼입 있어 모니터링 필요. |
| Warning | 위치·개수·감별 등에서 납득하기 어려운 패턴이 반복됨. 개선 후 재평가. |
| Critical | 체계적 오위치·대량 위양성 등으로 결과를 신뢰할 수 없음. |

## 2.4 처리 시간 적합성

알고리즘 처리 시간이 ARIA-H 진료 프로세스에 적합한지 확인

| 세부 항목 | 확인 사항 | 평가 방법 | 수집 대상 영상 |
|---|---|---|---|
| 단일 검사 분석 시간 | 한 검사(SWI/GRE) 처리 시간이 수용 범위인지 | PC 사양 명시 측정값 검토 | — |
| 부하 변동성 | 고해상도·slice 수 증가 시 처리 시간이 과도하게 늘지 않는지 | 부하 조건별 시간 분포 검토 | SWI 슬라이스 수 많은 case |
| 긴급도 적합성 | 정규 추적 및 의심 사례의 신속 확인 요구를 수용할 수 있는지 | 임상 프로세스 요구 시간 대비 검토 | — |

**등급 기준**

| 등급 | 기준 |
|---|---|
| Good | 진료 프로세스에 충분히 적합. 고부하 조건에서도 안정적. |
| Moderate | 대체로 적합하나 고부하 시 지연 발생. 운영상 주의 안내 필요. |
| Warning | 워크플로우에 부담을 주는 수준. 개선 후 재평가. |
| Critical | 처리 시간이 임상 활용을 불가능하게 할 정도로 과도함. |
